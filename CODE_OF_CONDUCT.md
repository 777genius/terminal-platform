# Code of Conduct

## Our Standard

We want Terminal Platform to be a professional, respectful, technically serious project.

Examples of behavior that contribute to a good environment:

- giving clear technical feedback
- challenging ideas without attacking people
- reporting bugs and risks precisely
- respecting differences in experience and background
- focusing discussions on facts, tradeoffs, and outcomes

Examples of unacceptable behavior:

- harassment or personal attacks
- hostile or insulting language
- doxxing or publishing private information
- bad-faith derailment of technical discussions
- repeated spam or disruptive behavior

## Scope

This code of conduct applies in:

- GitHub issues
- pull requests
- code review
- project discussions
- other public project communication spaces

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, or other contributions
that violate this code of conduct.

## Reporting

If you encounter unacceptable behavior, contact the repository owner through GitHub.
