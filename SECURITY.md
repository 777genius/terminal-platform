# Security Policy

## Reporting A Vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Use one of these paths instead:

- GitHub private vulnerability reporting, if it is enabled for the repository
- direct contact with the repository owner through GitHub

When reporting a vulnerability, include:

- affected component or crate
- affected platform and backend, if known
- reproduction steps
- expected impact
- whether the issue is local-only, embed-surface, or protocol-related

## Supported Versions

Terminal Platform is still in v1 release-candidate closeout.

That means:

- `main` is the primary supported line
- fixes will be prioritized on the current pre-v1 line
- older unpublished local states are not treated as supported releases
